**ALCOR 风控白皮书 v0.2**

（Algorithmic Control & Risk Oversight · Whitepaper v0.2）

**0\. 概述（Executive Summary）**

ALCOR 是一个面向银行、量化基金、资产管理公司的算法化风险监管系统，目标是实现：

* 风险可拆解（Decomposable）

* 风险可度量（Measurable）

* 风险可对冲（Hedgeable）

* 风险可追责（Auditable）

它不是一个“预测收益”的工具，而是一个“控制风险”的系统。

它的理念与 EDCA 的表达驱动、可验证链路一致，但不暴露任何内部协议，只体现“可审计性原则”。

**1\. ALCOR 核心哲学（The Philosophy of ALCOR）**

**1.1 风险来自三个维度**

| 风险类型 | 来源 | 典型表现 |
| :---- | :---- | :---- |
| 系统性风险 | 市场波动、宏观政策、宽松/收紧周期 | 宽基指数下跌、风险偏好收缩 |
| 行业因子风险 | 行业周期、景气上升/下行 | 行业板块轮动 |
| 特异性风险 | 个股事件、公司财务、管理层变化 | 暴雷、黑天鹅 |

一个健康的系统必须能回答：

👉 “组合今天的风险，来自哪里？”

👉 “如果明天再来一次黑天鹅，最大损失是多少？”

**3\. Module 1 — RME 风险度量层**

**3.1 多维 VaR（Value at Risk）**

5日 95% VaR：标准极值损失

压力测试 VaR：如“加息 50 bps 对组合影响”

**3.2 风格因子暴露（Style-Factor Exposure）**

监控如下常用因子暴露度：

* 价值 Value

* 动量 Momentum

* 成长 Growth

* 低波动 Low Vol

* ROE 因子

* 流动性因子

若某因子暴露超过历史 95% 分位数 → 黄色预警

超过 99% 分位数 → 红色预警

**3.3 风险上升的信号触发器**

* 风险偏好从 Risk-on → Risk-off

* 行业景气反转

* 组合波动率突破 2σ

**4\. Module 2 — DHO 动态对冲优化器**

核心目标：

👉 保持目标因子不变（如：仅吃价值因子的钱）

👉 剔除不需要的风险（如：动量、行业、市场 β）

**4.1 对冲工具库**

| 风险类型 | 建议对冲工具 |
| :---- | :---- |
| 系统性风险 | 指数期货 / ETF |
| 行业风险 | 行业 ETF / 行业期货 |
| 风格风险 | 风格 ETF（价值、动量、低波动） |
| 个股风险 | 做空单只股票 / 期权 |

**4.2 目标函数（核心）**

min⁡δσres²

subject to

Target Factor Exposure \= Constant

**5\. Module 3 — AAR 归因与审计层**

这是 ALCOR 最“杀穿机构”的模块。

**5.1 每日风控报告包括：**

组合总风险：xx%

其中：

市场 β 来源：40%

行业风险：35%

特异性风险：25%

**5.2 对冲有效性报告**

如果昨天未对冲：

\- 本金损失：-0.82%

实际损失：-0.14%

对冲节省损失：+0.68%

**5.3 智能审计链（与 EDCA 原则兼容但不暴露实现）**

* 每一个判断都有结构化理由

* 每一个结论都有可回溯路径

* 每一个对冲都能验证是否有效

**6\. 为什么银行必须采用 ALCOR？（Value Proposition）**

* ✔ 极大降低监管资本占用  
  因为 VaR 被压低。

* ✔ 使基金经理具备统一语言  
  风险沟通从“主观判断”变成“结构化事实”。

* ✔ 让黑箱变成玻璃箱  
  这就是机构最缺的东西。
